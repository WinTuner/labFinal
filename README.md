📚 City Central Library Management System
🎯 Overview

โปรเจคนี้เป็นระบบจัดการห้องสมุดที่พัฒนาด้วยภาษา Java โดยนำหลักการ Object-Oriented Programming (OOP) มาประยุกต์ใช้ในการออกแบบโครงสร้างโปรแกรมให้มีความเป็นระบบและสามารถขยายต่อได้ในอนาคต ตัวระบบรองรับการจัดการสมาชิก รายการหนังสือ และกระบวนการยืม-คืน พร้อมทั้งมีการจำลองสถานการณ์ใช้งานจริงผ่านเมนูแบบ command-line

✨ Features หลักของระบบ
👤 การจัดการสมาชิก (Member Management)

ระบบสามารถเพิ่มสมาชิกใหม่ ดูข้อมูลสมาชิก และค้นหาสมาชิกด้วยรหัสได้ โดยสมาชิกแต่ละคนสามารถเลือกประเภทสมาชิก (Membership) ได้ ซึ่งมีผลต่อจำนวนการยืม ระยะเวลาการยืม และส่วนลดค่าปรับ

📖 การจัดการทรัพยากร (Library Items)

รองรับสื่ออย่างน้อย 2 ประเภท ได้แก่ หนังสือแบบเล่ม (Physical Book) และหนังสือดิจิทัล (E-Book) โดยสามารถเพิ่ม ค้นหา แสดงผล และลบรายการได้ พร้อมแสดงสถานะว่าว่างหรือถูกยืมอยู่

🔁 ระบบยืม-คืน (Borrowing System)

ผู้ใช้สามารถยืมและคืนรายการได้จริง โดยระบบจะตรวจสอบเงื่อนไข เช่น

รายการว่างหรือไม่
จำนวนการยืมของสมาชิก
ระยะเวลาการยืมตามประเภทสมาชิก

นอกจากนี้ยังมีการคำนวณค่าปรับกรณีคืนล่าช้า

📊 รายงาน (Reports)

ระบบสามารถแสดงรายงานพื้นฐาน ได้แก่

รายการที่ค้างส่ง (Overdue)
รายการที่ถูกยืมบ่อย
สมาชิกที่มีการยืมสูงสุด
สถานะของรายการทั้งหมด
💾 การจัดเก็บข้อมูล (Data Persistence)

ข้อมูลของระบบจะถูกจัดเก็บในรูปแบบไฟล์ CSV โดยสามารถโหลดข้อมูลเดิมกลับมาใช้งานได้เมื่อเปิดโปรแกรม และบันทึกข้อมูลอัตโนมัติเมื่อออกจากระบบ

🧠 การประยุกต์ใช้ OOP

โปรเจคนี้มีการนำแนวคิด OOP มาใช้ครบถ้วน ดังนี้:

Inheritance: คลาส PhysicalBook และ EBook สืบทอดจาก LibraryItem
Polymorphism: ใช้ LibraryItem เป็นตัวแทนของหลายประเภทสื่อ
Interface: ใช้ MembershipStrategy, Taxable, และ DigitalContent
Overriding: override เมธอด เช่น displayDetails()
Overloading: เมธอด calculateLateFee() รองรับหลายรูปแบบ
Encapsulation: ใช้ private fields และ getter/setter เพื่อควบคุมข้อมูล

นอกจากนี้ยังมีการใช้ Design Patterns เช่น

Singleton Pattern สำหรับ LibrarySystem
Strategy Pattern สำหรับระบบ Membership

🎯 สรุป

ระบบนี้สามารถทำงานได้จริงตามเงื่อนไขของโจทย์ มีการใช้ OOP อย่างครบถ้วน และมีโครงสร้างที่ชัดเจน รองรับการใช้งานในสถานการณ์จำลองได้อย่างเหมาะสม รวมถึงสามารถต่อยอดเพิ่มเติมฟีเจอร์ในอนาคตได้ เช่น การพัฒนา GUI หรือระบบจองหนังสือ
